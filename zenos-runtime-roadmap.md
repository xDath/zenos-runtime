# Etla Runtime v1 Roadmap

**Status:** Canonical roadmap  
**Canonical specification:** `docs/etla-runtime-v1-roadmap.md`

Etla Runtime v1 evolves Zenos Runtime into a token-efficient, Codex-class execution ecosystem.

Its primary goals are:

- make lower-tier models complete most routine work;
- minimize premium Boss calls and premium-token usage;
- use tools and deterministic validation as the source of truth;
- compile role-specific context instead of dumping raw inputs;
- preserve Zenos Memory as a full-cloud service while adding backward-compatible Runtime capabilities;
- move builds, calibration, evaluation, and other heavy workloads away from the VPS to GitHub Actions and Vercel.

```text
Lower-tier models = primary workforce
Tools             = source of truth
Runtime           = manager and execution authority
Verifier          = quality control
Boss              = rare premium judgment
Zenos Memory      = cloud continuity and durable knowledge
Runtime Lab       = heavy cloud evaluation and optimization
```

## Implementation Sequence

1. Compatibility, authentication, and VPS resource protection.
2. Token Economy Engine and Boss Token Minimizer.
3. Runtime Context Compiler.
4. Repository Intelligence and Tool Broker.
5. Codex-class coding state machine and remote validation.
6. Skill ecosystem.
7. Model calibration and capability profiles.
8. Adaptive routing and prompt optimization.
9. Intelligence Bridge with Zenos Memory.
